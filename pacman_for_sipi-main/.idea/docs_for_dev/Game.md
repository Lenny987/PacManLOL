# 🟡 Класс `Game`
📁 `main.py` | 🏗️ Ядро приложения / Игровой цикл

## 📋 Назначение
Центральный класс игры, реализующий основной игровой цикл и управляющий всеми состояниями приложения. Выступает точкой входа и координатором взаимодействия между другими модулями.

## 🧩 Атрибуты
| Атрибут | Тип | Описание |
|---------|-----|----------|
| `running` | `bool` | Флаг работы приложения |
| `state` | `str` | Текущее состояние (`menu`/`game`/`pause`/`editor`/`leaderboard`/`game_over`) |
| `fps` | `int` | Частота кадров (по умолчанию `60`) |
| `screen` | `pygame.Surface` | Основная поверхность экрана |
| `clock` | `pygame.time.Clock` | Объект для контроля FPS |
| `player` | `Player` | Экземпляр управляемого персонажа |
| `ghosts` | `list[Ghost]` | Список активных призраков |
| `level` | `Level` | Текущий загруженный уровень |
| `score` | `int` | Текущий счёт игрока |

## ⚙️ Методы
| Метод | Описание |
|-------|----------|
| `__init__()` | Инициализация игры, загрузка настроек, создание зависимостей |
| `run()` | Запуск главного игрового цикла |
| `handle_events()` | Обработка событий `pygame` |
| `update()` | Обновление состояния игровых объектов |
| `draw()` | Отрисовка текущего кадра |
| `change_state(new_state)` | Переключение состояния игры |
| `reset_game()` | Сброс игры к начальным условиям |
| `quit()` | Корректное завершение приложения |

## 🔗 Взаимодействие
- **Зависит от:** `Settings`, `SoundManager`, `Player`, `Ghost`, `Level`, `Menu`, `LevelEditor`
- **Управляет:** жизненным циклом приложения и переходом между экранами.
